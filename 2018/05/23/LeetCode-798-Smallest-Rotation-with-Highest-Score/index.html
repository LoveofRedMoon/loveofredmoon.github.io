LeetCode 798 Smallest Rotation with Highest Score

题目

Given an array A, we may rotate it by a non-negative integer K so that the array becomes A[K], A[K+1], A{K+2], … A[A.length - 1], A[0], A[1], …, A[K-1]. Afterward, any entries that are less than or equal to their index are worth 1 point.

For example, if we have [2, 4, 1, 3, 0], and we rotate by K = 2, it becomes [1, 3, 0, 2, 4]. This is worth 3 points because 1 > 0 [no points], 3 > 1 [no points], 0 <= 2 [one point], 2 <= 3 [one point], 4 <= 4 [one point].

Over all possible rotations, return the rotation index K that corresponds to the highest score we could receive. If there are multiple answers, return the smallest such index K.

Example 1:
Input: [2, 3, 1, 4, 0]
Output: 3
Explanation:
Scores for each K are listed below:
K = 0, A = [2,3,1,4,0], score 2
K = 1, A = [3,1,4,0,2], score 3
K = 2, A = [1,4,0,2,3], score 3
K = 3, A = [4,0,2,3,1], score 4
K = 4, A = [0,2,3,1,4], score 3
So we should choose K = 3, which has the highest score.

思路

在每一次左移的过程中, 除了首位, 各个索引均降低 1 首位索引提高 A.length

因此

  1. 先过滤所有的 0, 避免出现首位为 0 的情况, 避免对位置的影响, 应当把 0 改为很大的数, 例如 A.length
    • 因为 0<=POS 恒成立, 因此一定是不会改变 point 的
  2. 先处理所有值与索引的差
    • 例子中是 [2, 2, -1, 1, -4]
  3. 在左移一次后, 所有值加一, 除了(移动的次数位置-1)减少 A.length
    • 例子中是 K=1 [-2, 3, 0, 2, -3]
    • K=2 [-2, -2, 1, 3, -2]

总结: K=0 单独处理
