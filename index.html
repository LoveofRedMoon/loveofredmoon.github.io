懒加载实现原理

懒加载

懒加载是优化用户体验的一步, 并且已经是个很古老的问题了.
这里就懒加载的一些情况进行说明

图片懒加载

一开始肯定是想使用默认图片, 并且加载完成之后替换原本的 SRC 即可但是事与愿违, 图片修改 SRC 之后会出现空白

因此使用情况应当是使用一个新的 IMG 标签加载图片, 如果加载完成, 说明浏览器有缓存了, 这个时候替换原来的图片即可瞬间加载完毕. 当然, 为了保持文件大小的一致性, 一定要给 Img 标签设置具体的宽高值

加载时机

上面的加载时机是自己决定的所以一般是使用window.onload即整个网页架构都加载完毕后进行图片加载具体的图片即

但是在实际开发中, 会遇到很多问题, 例如淘宝, 在用户对界面很熟悉的情况下, 一般会忽略打开页面的顶部信息栏, 而是直接滚动到下部选择需要的物品又或者用户会快速浏览商品名, 对某些商品进行过滤

因此需要快速进行定位一般来说, 需要两步同时进行,
即预加载图片和界面加载图片同时进行

我的思路里, 认定当前界面的图片加载 > 下一屏加载 > 其他图片加载

  1. 图片位置判定是利用元素距离顶部的距离 视口距离顶部的距离 视口的高度三者的关系来判定图片的位置即若在视口内, 则是$('img').offset().top - $(window).scrollTop()必须在0和$(window).height之间, 若是下一屏则是在$(window).height和$(window).height*2之间
  2. 时机的判定应当在加载完 DOM 结构之后就进行网页图片的申请, 并且需要进行图片的管理

关于一些常见类型的判断

JS 的数据类型

JS 基本上只分为
number string undefined boolean object symbol
当然, 从object里面可以拓展出一些其他类型

基础数据类型判断

  1. number
    判断数字很简单, typeof a === 'number' 即可,
    在此基础上, 衍生出一些其他判断以下方法在全局下会进行隐式转换, 例如"123"也被认为是数字
    Number.isNaN(a) 判断是否(不是)一个数字
    Number.isFinite(a) 判断是否是个有穷数(即不包括Infinite)
    Number.isInterger(a) 判断是否是整数, 没有全局方法额外要注意的是typeof NaN === 'number'是true

  2. string
    同样, 直接typeof a === 'string' 即可

  3. undefined
    undefined仅有undefined一个值, 直接判断a === undefined即可

  4. Boolean
    typeof a === 'boolean'

  5. symbol
    typeof a === 'symbol'

  6. object
    由于历史原因, 需要区分null和object
    由于null是falsy, 因此可以用if(a)来判断是否是object
    因此具体判断应该是if(a && typeof a === 'object')
    具体需要具体的类

  7. Element
    直观理解的方法就是a instanseof Element即可判断,
    问题出在IE7, 没有Element, 那么我们使用if(a.nodeType)来判断是否是节点, 或判断节点的具体类型

  8. Array
    Array.isArray(a)

  9. 其他暂未想到, 可以考虑使用 instanceof 暂时代替

LeetCode 75 分类颜色

题目

给定一个包含红色、白色和蓝色，一共 n 个元素的数组，原地对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。

此题中，我们使用整数 0、 1 和 2 分别表示红色、白色和蓝色。

注意:
不能使用代码库中的排序函数来解决这道题。

思路

使用递归的思想解决, 假设当前位置之前的均已经是排好序的列表

使用标记标记出当前 0 1 的最后位置,

当前位置可以分为以下几种情况例如 0 0 1 1 2 2 1
如果是 0, 就和 0 后一个位置交换, 0 的位置加一 (并且继续判断, 因为可能换回来的是 1 或者 2)
如果是 1 就和 1 后面一个位置交换, 1 的位置加一 (此时换回来的一定是 2 (除非位置没变,即没有 2))
如果是 2 不动

防止出现没有 0 1 2 而造成的序号混乱的情况, 先往数组中加入 0 1 2 三个数, 最后删除这样

在处理 0 之后一定要处理 1, 不用进行判断了

LeetCode 809 情感丰富的文字

题目

有时候人们会用额外的字母来表示额外的情感，比如 “hello” -> “heeellooo”, “hi” -> “hiii”。我们将连续的相同的字母分组，并且相邻组的字母都不相同。我们将一个拥有三个或以上字母的组定义为扩张状态（extended），如第一个例子中的 “e” 和” o” 以及第二个例子中的 “i”。 此外，”abbcccaaaa” 将有分组 “a” , “bb” , “ccc” , “dddd”；其中 “ccc” 和 “aaaa” 处于扩张状态。

对于一个给定的字符串 S ，如果另一个单词能够通过将一些字母组扩张从而使其和 S 相同，我们将这个单词定义为可扩张的（stretchy）。我们允许选择一个字母组（如包含字母 c ），然后往其中添加相同的字母 c 使其长度达到 3 或以上。注意，我们不能将一个只包含一个字母的字母组，如 “h”，扩张到一个包含两个字母的组，如 “hh”；所有的扩张必须使该字母组变成扩张状态（至少包含三个字母）。

输入一组单词，输出其中可扩张的单词数量。

思路

将单词分割成各个字母部分, 并且标记允许的个数例如 h => 只能是 h
hh => 只能是 hh
hhh => 可以是 h 或者 hhh
hhhh => 可以是 h hh 或者 hhhh
因此 个数不等于自己个数减一即可, 并且大于 0

LeetCode 295 数据流的中位数

题目

中位数是有序列表中间的数。如果列表长度是偶数，中位数则是中间两个数的平均值。

例如，

[2,3,4] 的中位数是 3

[2,3] 的中位数是 (2 + 3) / 2 = 2.5

设计一个支持以下两种操作的数据结构：

void addNum(int num) - 从数据流中添加一个整数到数据结构中。
double findMedian() - 返回目前所有元素的中位数。

思路

这个思想对于 JS 程序猿应该不是问题, 因为继承和原型是一个老生常谈的东西了重点在于保存的数据结构, 能够快速读取中间数

第一种思路是直接保存住中间数的位置第二种思路是将数组分成两半进行保存显然第二种思路是不太优秀的, 因为数据插入时可能导致两个数组的元素进行频繁交换

因此这里采用了第一种思路, 并且使用位置保存住中位数的位置

LeetCode 722 删掉注释

题目

在 C++ 中有两种注释风格，行内注释和块注释。

字符串// 表示行注释，表示//和其右侧的其余字符应该被忽略。

字符串/_ 表示一个块注释，它表示直到_/的下一个（非重叠）出现的所有字符都应该被忽略。（阅读顺序为从左到右）非重叠是指，字符串/*/并没有结束块注释，因为注释的结尾与开头相重叠。

第一个有效注释优先于其他注释：如果字符串//出现在块注释中会被忽略。 同样，如果字符串/*出现在行或块注释中也会被忽略。

如果一行在删除注释之后变为空字符串，那么不要输出该行。即，答案列表中的每个字符串都是非空的。

样例中没有控制字符，单引号或双引号字符。比如，source = “string s = “/_ Not a comment. _/“;” 不会出现在测试样例里。（此外，没有其他内容（如定义或宏）会干扰注释。）

我们保证每一个块注释最终都会被闭合， 所以在行或块注释之外的/*总是开始新的注释。

最后，隐式换行符可以通过块注释删除。 有关详细信息，请参阅下面的示例。

从源代码中删除注释后，需要以相同的格式返回源代码。

思路

在未匹配状态时, 寻找// 或/*
如果是 // 选择忽略从//往后的所有字符如果是 /* 进入匹配状态, 并且从当前位置继续开始匹配同时记录 该行行号以及位置在匹配状态时, 寻找/,
如果找到 则获取/ 之后的字符, 并且匹配到/*所在行, 并且从当前行重新匹配如果未找到 则删除整行

LeetCode 843 猜猜这个单词

题目

这个问题是 LeetCode 平台新增的交互式问题 。

我们给出了一个由一些独特的单词组成的单词列表，每个单词都是 6 个字母长，并且这个列表中的一个单词将被选作秘密。

你可以调用 master.guess(word) 来猜单词。你所猜的单词应当是存在于原列表并且由 6 个小写字母组成的类型字符串。

此函数将会返回一个整型数字，表示你的猜测与秘密单词的准确匹配（值和位置同时匹配）的数目。此外，如果你的猜测不在给定的单词列表中，它将返回 -1。

对于每个测试用例，你有 10 次机会来猜出这个单词。当所有调用都结束时，如果您对 master.guess 的调用不超过 10 次，并且至少有一次猜到秘密，那么您将通过该测试用例。

除了下面示例给出的测试用例外，还会有 5 个额外的测试用例，每个单词列表中将会有 100 个单词。这些测试用例中的每个单词的字母都是从 ‘a’ 到 ‘z’ 中随机选取的，并且保证给定单词列表中的每个单词都是唯一的。

这道题十分有意思, 等下先把 guess 函数给出(虽然很简单)

根据该字母出现的频率, 那么给予权重 (频率-1/2)^2 这样越接近 1/2 的字母就越有可能被选中为什么是 1/2 呢? 因为在查找时, 二分法永远是最快的,每次肯定筛选一半

然后利用 guess 去筛选下一次的数组
