LeetCode 826 Most Profit Assigning Work

题目

We have jobs: difficulty[i] is the difficulty of the ith job, and profit[i] is the profit of the ith job.

Now we have some workers. worker[i] is the ability of the ith worker, which means that this worker can only complete a job with difficulty at most worker[i].

Every worker can be assigned at most one job, but one job can be completed multiple times.

For example, if 3 people attempt the same job that pays $1, then the total profit will be $3. If a worker cannot complete any job, his profit is $0.

What is the most profit we can make?

思路

想实现很简单, 直接将收益排序, 让员工选择收益最大的且自己能完成的项目即可

LeetCode 565 数组嵌套

题目

索引从 0 开始长度为 N 的数组 A，包含 0 到 N - 1 的所有整数。找到并返回最大的集合 S，S[i] = {A[i], A[A[i]], A[A[A[i]]], … }且遵守以下的规则。

假设选择索引为 i 的元素 A[i]为 S 的第一个元素，S 的下一个元素应该是 A[A[i]]，之后是 A[A[A[i]]]… 以此类推，不断添加直到 S 出现重复的元素。

思路

本质上是数学中群的概念.

LeetCode 3 无重复字符的最长子串

题目

给定一个字符串，找出不含有重复字符的最长子串的长度。

示例：

给定 “abcabcbb” ，没有重复字符的最长子串是 “abc” ，那么长度就是 3。

给定 “bbbbb” ，最长的子串就是 “b” ，长度是 1。

给定 “pwwkew” ，最长子串是 “wke” ，长度是 3。请注意答案必须是一个子串，”pwke” 是 子序列 而不是子串。

思路

对于一个字符串, 依次判断每个字符出现的次数, 如果出现超过 1 次, 那么就进行分割, 并且分别再次求子字符串的最长子串

LeetCode 798 Smallest Rotation with Highest Score

题目

Given an array A, we may rotate it by a non-negative integer K so that the array becomes A[K], A[K+1], A{K+2], … A[A.length - 1], A[0], A[1], …, A[K-1]. Afterward, any entries that are less than or equal to their index are worth 1 point.

For example, if we have [2, 4, 1, 3, 0], and we rotate by K = 2, it becomes [1, 3, 0, 2, 4]. This is worth 3 points because 1 > 0 [no points], 3 > 1 [no points], 0 <= 2 [one point], 2 <= 3 [one point], 4 <= 4 [one point].

Over all possible rotations, return the rotation index K that corresponds to the highest score we could receive. If there are multiple answers, return the smallest such index K.

Example 1:
Input: [2, 3, 1, 4, 0]
Output: 3
Explanation:
Scores for each K are listed below:
K = 0, A = [2,3,1,4,0], score 2
K = 1, A = [3,1,4,0,2], score 3
K = 2, A = [1,4,0,2,3], score 3
K = 3, A = [4,0,2,3,1], score 4
K = 4, A = [0,2,3,1,4], score 3
So we should choose K = 3, which has the highest score.

思路

在每一次左移的过程中, 除了首位, 各个索引均降低 1 首位索引提高 A.length

因此

  1. 先过滤所有的 0, 避免出现首位为 0 的情况, 避免对位置的影响, 应当把 0 改为很大的数, 例如 A.length
    • 因为 0<=POS 恒成立, 因此一定是不会改变 point 的
  2. 先处理所有值与索引的差
    • 例子中是 [2, 2, -1, 1, -4]
  3. 在左移一次后, 所有值加一, 除了(移动的次数位置-1)减少 A.length
    • 例子中是 K=1 [-2, 3, 0, 2, -3]
    • K=2 [-2, -2, 1, 3, -2]

总结: K=0 单独处理

当 K>0 时, 第 i 个位置的差值应当为 (i<K?-A.length:0) + K + cur

LeetCode 393 UTF-8 编码验证

题目

UTF-8 中的一个字符可能的长度为 1 到 4 字节，遵循以下的规则：

对于 1 字节的字符，字节的第一位设为 0，后面 7 位为这个符号的 unicode 码。对于 n 字节的字符 (n > 1)，第一个字节的前 n 位都设为 1，第 n+1 位设为 0，后面字节的前两位一律设为 10。剩下的没有提及的二进制位，全部为这个符号的 unicode 码。这是 UTF-8 编码的工作方式：

Char. number range | UTF-8 octet sequence
(hexadecimal) | (binary)
——————–+———————————————
0000 0000-0000 007F | 0xxxxxxx
0000 0080-0000 07FF | 110xxxxx 10xxxxxx
0000 0800-0000 FFFF | 1110xxxx 10xxxxxx 10xxxxxx
0001 0000-0010 FFFF | 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx
给定一个表示数据的整数数组，返回它是否为有效的 utf-8 编码。

思路

  1. 若为 1 字节, 则首位为 0, 即小于 128
  2. 若为多字节, 则先根据收个判断是否是合法的多字节,
    • 192-223 2 字节
    • 224-239 3 字节
    • 240-247 4 字节
  3. 根据前面的字节数 来获取后面是否合乎规范 128-191
